You also have .node-version → 20 pinned in the backend, but the build is using Node 22.23.1 because Railpack ignores .node-version when the root directory is set. Railpack looks for version files only at the repo root, not in the subdirectory.
Why npm ci Failed

When Railpack sets rootDirectory: mevrelbank/backend, it:

    Changes into that directory
    Runs npm ci with the backend's package-lock.json (70 KB, complex)
    Uses Node 22.23.1 from the repo root (not the pinned 20)

The mismatch between Node 22.23.1 and a lockfile built for Node 20 + npm version misalignment triggered the npm ci crash.

Create .node-version at the repo root with the same pinned version:
