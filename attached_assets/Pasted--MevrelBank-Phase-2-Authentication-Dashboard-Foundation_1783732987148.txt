# Implementation Plan

## Goal

Implement a fully functional client-side authentication system (Phase 2) and extract the Customer Banking Portal dashboard (Phase 3) from the mock design system into a routed page at `/dashboard`.

This allows visitors to:

- Register
- Verify email
- Sign in
- Complete MFA
- Access the dashboard

All state should be dynamically persisted via localStorage.

---

## Authentication Context & State

### NEW — AuthContext.tsx

Create an authentication context that manages:

- user
- isAuthenticated
- isMfaRequired
- tempUser

Persist:

- current session
- registered users

using localStorage.

Provide mock implementations for:

- login(email, password)
- register(name, email, password, accountType)
- verifyOTP(code)
- verifyMFA(code)
- logout()

---

### NEW — ProtectedRoute.tsx

Create a protected route helper.

Behaviour:

- Redirect unauthenticated users attempting to access `/dashboard` to `/login`.
- Redirect authenticated users attempting to access `/login` or `/register` back to `/dashboard`.
